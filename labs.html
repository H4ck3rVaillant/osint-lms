🧪 Lab OSINT – Reconnaissance passive d’une organisation

🎯 Objectif du lab

Réaliser une reconnaissance OSINT passive et légale afin d’identifier les points d’exposition informationnelle d’une organisation.

⚖️ Cadre légal

  • Données accessibles publiquement
  • Aucune interaction active avec les systèmes
  • Cible autorisée (lab, domaine personnel, entreprise publique)

🗂️ Phase 1 – Définition de la cible

Action :

  • Identifier le domaine principal
  • Identifier l’activité de l’organisation
  • Identifier sa présence publique

Livrable :

Organisation :
Domaine :
Secteur :

🌐 Phase 2 – Recherche de services exposés (Shodan)

Action :

hostname:example.com

Données à collecter :

  • Adresses IP exposées
  • Ports ouverts
  • Services visibles

Analyse attendue :

  • Service inutile exposé ?
  • Version obsolète ?
  • Surface d’attaque visible ?

🧩 Phase 3 – OSINT automatisé (Recon-ng)

Commandes Kali :

recon-ng
workspaces create lab_osint
workspaces select lab_osint
add domains example.com

Modules recommandés :

modules load recon/domains-contacts/whois_pocs
run

Données collectées :

  • Emails publics
  • Contacts WHOIS

📧 Phase 4 – Emails & sous-domaines (theHarvester)

Commandes Kali :

theHarvester -d example.com -b google
theHarvester -d example.com -b crtsh

Données collectées :

  • Emails professionnels
  • Sous-domaines exposés

📊 Phase 5 – Analyse et synthèse

Catégorie Donnée Risque potentiel

Sous-domaine dev.example.com Environnement exposé

✅ Critères de réussite

  • Données organisées et sourcées
  • Analyse argumentée
  • Aucune action intrusive

🎯 Exploitation professionnelle

  • Portfolio OSINT

📋 Checklist d’audit OSINT – Reconnaissance passive

Cette checklist permet de vérifier qu’une reconnaissance OSINT passive a été menée de manière structurée, légale et exploitable.

🗂️ 1. Définition du scope

  • Domaine principal identifié
  • Organisation clairement définie
  • Cible autorisée (publique / lab / perso)
  • Aucune interaction intrusive prévue

🌐 2. Recherche via moteurs & Shodan

  • Recherche Shodan effectuée
  • IP exposées identifiées
  • Ports ouverts recensés
  • Services et versions notés
  • Exposition potentielle analysée

🧩 3. OSINT automatisé – Recon-ng

  • Workspace créé
  • Domaine ajouté
  • Modules OSINT exécutés
  • Emails publics collectés
  • Contacts WHOIS analysés

📧 4. Emails & sous-domaines – theHarvester

  • theHarvester exécuté
  • Emails professionnels identifiés
  • Sous-domaines collectés
  • Rapport HTML généré

📊 5. Analyse & corrélation

  • Données croisées entre outils
  • Patterns identifiés (emails, infra)
  • Surface d’attaque visible évaluée
  • Aucun faux positif évident

📄 6. Livrables

  • Notes OSINT structurées
  • Tableau de synthèse complété
  • Rapport prêt pour client / portfolio

⚖️ 7. Conformité légale

  • Sources publiques uniquement
  • Pas de scan actif
  • Pas d’exploitation

🎯 Validation finale

Si toutes les cases sont cochées, la phase OSINT est :

  • ✔️ complète
  • ✔️ légale
  • ✔️ exploitable professionnellement
