🧪 Lab 1 – Reconnaissance OSINT passive

🎯 Objectif du lab

Réaliser une mission OSINT complète, 100 % passive et légale, afin d’identifier l’exposition informationnelle d’une organisation fictive.

⚖️ Cadre légal

  • Données publiques uniquement
  • Aucune interaction intrusive
  • Aucun scan actif
  • Domaine de test autorisé

🧭 Contexte

Vous êtes analyste OSINT junior. Votre mission consiste à analyser l’exposition publique d’une organisation fictive nommée ExampleCorp.

🔍 Phase 1 – Définition de la cible

  • Domaine principal : example.com
  • Activité : entreprise fictive
  • Type d’OSINT : passif

🌐 Phase 2 – Recherche via Shodan

Accéder à Shodan et utiliser les requêtes suivantes :

hostname:example.com
hostname:example.com port:443

À collecter :

  • Adresses IP visibles
  • Ports exposés
  • Services identifiés

🧩 Phase 3 – Recon-ng (Kali Linux)

Lancer Recon-ng :

recon-ng

Créer un workspace :

workspaces create lab_osint

Ajouter le domaine :

add domains example.com

Installer un module :

marketplace install recon/domains-hosts/hackertarget

Lancer le module :

modules load recon/domains-hosts/hackertarget
run

📧 Phase 4 – theHarvester

Depuis Kali :

theHarvester -d example.com -b crtsh

À collecter :

  • Sous-domaines
  • Emails publics éventuels

🧠 Phase 5 – Analyse

Interpréter les données collectées :

  • Pourquoi ces informations sont-elles visibles ?
  • Quel risque informationnel potentiel ?
  • Que pourrait améliorer l’organisation ?

📄 Phase 6 – Restitution

Rédiger une synthèse OSINT contenant :

  • Objectif
  • Méthodologie
  • Résultats
  • Analyse
  • Recommandations

✅ Critères de réussite

  • Données correctement collectées
  • Analyse claire et argumentée
  • Aucune action intrusive
  • Rapport structuré

📋 Checklist d’audit OSINT – Reconnaissance passive

Cette checklist permet de structurer une mission OSINT 100 % passive dans un cadre légal et professionnel.

🧭 1. Cadrage de la mission

  • ☐ Objectif clairement défini
  • ☐ Périmètre autorisé validé
  • ☐ Type d’OSINT : passif uniquement
  • ☐ Domaine(s) autorisé(s)

🌐 2. Recherche générale (Open Web)

  • ☐ Recherche Google classique
  • ☐ Google Dorks utilisés
  • ☐ WHOIS consulté
  • ☐ Présence sur réseaux sociaux

🔍 3. Exposition réseau (Shodan)

  • ☐ Recherche par domaine
  • ☐ Recherche par IP
  • ☐ Ports exposés identifiés
  • ☐ Services visibles
  • ☐ Versions logicielles notées

🧩 4. Collecte automatisée (Recon-ng)

  • ☐ Workspace créé
  • ☐ Domaine ajouté
  • ☐ Modules OSINT installés
  • ☐ Données collectées (hosts, IPs)

📧 5. Emails & sous-domaines

  • ☐ theHarvester exécuté
  • ☐ Sous-domaines identifiés
  • ☐ Emails publics recensés

🧠 6. Corrélation & analyse

  • ☐ Données regroupées
  • ☐ Relations identifiées
  • ☐ Visualisation (Maltego)
  • ☐ Risques informationnels évalués

📄 7. Reporting

  • ☐ Résultats synthétisés
  • ☐ Analyse compréhensible
  • ☐ Recommandations formulées
  • ☐ Rapport structuré

⚖️ 8. Conformité légale & éthique

  • ☐ Données publiques uniquement
  • ☐ Aucune action intrusive
  • ☐ Respect du Responsible Disclosure

✅ Validation finale
