📚 Module 2 – Shodan : analyse de l’exposition publique

🎯 Objectifs pédagogiques

  • Comprendre à quoi sert Shodan dans une démarche OSINT
  • Savoir lire et interpréter les résultats
  • Identifier une exposition inutile ou risquée

🧠 À quoi sert Shodan ?

Shodan est un moteur de recherche spécialisé qui indexe les services exposés sur Internet (serveurs web, bases de données, caméras, VPN, etc.).

Contrairement à Google, Shodan ne cherche pas des pages web, mais des services techniques.

En OSINT, Shodan sert à :

  • Cartographier la surface d’exposition d’une organisation
  • Identifier des services accessibles publiquement
  • Détecter des versions obsolètes ou mal configurées

⚖️ Cadre légal

L’utilisation de Shodan est légale tant que :

  • Tu restes en consultation passive
  • Tu n’interagis pas avec les services
  • Tu respectes le scope défini

🛠️ Mise en pratique – premières recherches

1. Créer un compte gratuit

  • Lien : https://www.shodan.io

2. Recherche par domaine

domain:example.com

Cette recherche permet de voir :

  • les IP associées au domaine

    🎯 Objectifs pédagogiques

    • Comprendre le rôle de Recon-ng dans une démarche OSINT

    🧠 Présentation de Recon-ng

    Recon-ng est un framework OSINT en ligne de commande, inspiré de Metasploit. Il permet de structurer, automatiser et documenter une collecte d’informations à partir de sources ouvertes.

    Il est particulièrement utilisé pour :

    • OSINT corporate
    • pré-audit de sécurité
    • cartographie d’attaque passive

    ⚖️ Cadre légal

    Recon-ng est légal si :

    • les modules utilisés exploitent uniquement des sources publiques
    • aucune interaction intrusive n’est effectuée
    • le scope est clairement défini

    🛠️ Installation et lancement (Kali Linux)

    sudo apt update && sudo apt install recon-ng

    Lancer Recon-ng :

    📚 Module 4 – theHarvester : collecte d’emails et sous-domaines

    🎯 Objectifs pédagogiques

    • Identifier les emails exposés publiquement
    • Découvrir les sous-domaines d’une organisation
    • Comprendre l’exposition informationnelle d’une cible

    🧠 Présentation de theHarvester

    theHarvester est un outil OSINT en ligne de commande permettant de collecter des informations publiques via des moteurs de recherche et des bases ouvertes.

    Il est utilisé principalement en :

    • OSINT corporate
    • pré-audit de sécurité
    • cartographie d’attaque passive

    ⚖️ Cadre légal

    theHarvester est légal si :

    • la cible est autorisée (entreprise, lab, domaine personnel)
    • les sources interrogées sont publiques
    • aucune tentative d’exploitation n’est réalisée

    🛠️ Installation (Kali Linux)

    sudo apt update && sudo apt install theharvester

    🚀 Commande de base

    Syntaxe générale :

    theHarvester -d example.com -b google
    • -d : domaine cible
    • -b : source (google, bing, duckduckgo, crtsh, etc.)

    🔍 Sources recommandées

    theHarvester -d example.com -b google theHarvester -d example.com -b bing theHarvester -d example.com -b crtsh

    crtsh est très puissant pour découvrir des sous-domaines via les certificats TLS.

    📄 Génération d’un rapport HTML

    theHarvester -d example.com -b google -f rapport_theharvester.html

    Le rapport généré peut être intégré directement dans ton portfolio.

    📊 Types de données collectées

    • emails professionnels
    • sous-domaines exposés
    • hôtes liés à l’organisation

    🧠 Analyse OSINT (raisonnement attendu)

    Tu dois te poser les questions suivantes :

    • Les emails suivent-ils un pattern (prenom.nom) ?
    • Les sous-domaines révèlent-ils des environnements internes ?
    • Certains services semblent-ils obsolètes ou exposés ?

    🧪 Exercice pratique

    1. Choisir un domaine autorisé (lab ou domaine personnel)
    2. Lancer theHarvester avec 2 sources différentes
    3. Exporter un rapport HTML
    4. Identifier au moins :
      • 2 emails
      • 2 sous-domaines

    ✅ Solution attendue

    • Commandes correctement exécutées
    • Données structurées
    • Analyse orientée exposition et risques
